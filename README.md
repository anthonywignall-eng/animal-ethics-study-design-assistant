# Animal Ethics Study Design Assistant - MTD Module

A Streamlit prototype for designing and documenting an in vivo maximum tolerated dose (MTD) or tolerability study involving exatecan, SN-38, related prodrugs, or antibody-drug conjugates (ADCs).

This tool assists with study design and draft preparation only. It does not constitute ethics approval, guarantee Animal Ethics Committee approval, replace investigator judgement, replace veterinary advice, or replace Animal Ethics Committee review.

## What the app does

- Uses a shorter four-stage questionnaire to collect the main study-design drivers.
- Keeps detailed protocol fields in optional expanders so users can reach the recommendation faster.
- Applies transparent, rule-based logic to recommend a study pathway.
- Displays a study-design flowchart based on supplied answers.
- Flags missing critical information and design considerations.
- Generates animal ethics procedure considerations, including likely procedure categories, associated pain/distress considerations, anaesthesia/analgesia points requiring confirmation, welfare costs, animal timeline, and 3Rs content.
- Creates a downloadable Word report for internal draft preparation.
- Generates editable introduction and aims wording from rules and supplied answers only.

## What the app does not do

- It does not approve an ethics application.
- It does not provide veterinary, regulatory, or ethics approval advice.
- It does not recommend numerical dose levels.
- It does not use ChatGPT, OpenAI, external AI or external APIs.
- It does not invent animal numbers, tumour models, welfare thresholds, procedures or references.
- It does not decide that CO2, anaesthesia, blood collection or any other procedure is approved; these must be checked against institutional SOPs, veterinary advice and AEC requirements.
- It does not save study information to a database.
- It does not include login, user accounts, or persistent records in Version 1.

## Quiz structure

The faster workflow has four stages:

1. Core Aim and Test Article
2. Compound or ADC Readiness
3. Model, Route and Schedule
4. Welfare and Animal-use Essentials

The app can produce a recommendation from the core fields, while optional expanders let a user add extra detail for a richer Word report.

## How recommendations are made

The study pathway is generated by deterministic rules in `study_recommender.py` and `rules.py`. This keeps the recommendation auditable and easier to review. The report introduction and aims are rule-based draft wording that users can edit before downloading the Word report.

Example recommended pathways include:

- Preliminary tolerability or feasibility study before full MTD design
- Single-dose staged dose-escalation tolerability study
- Staged repeat-dose tolerability study with escalation-stop review
- Repeat-dose tolerability study aligned to downstream schedule
- Staged candidate-comparison tolerability design
- ADC-readiness confirmation followed by staged tolerability design
- Clarify missing critical information before finalising MTD design

## Files in this prototype

- `app.py` - main Streamlit application.
- `questionnaire.py` - four-stage questionnaire rendering and conditional question logic.
- `config.py` - app text, stage definitions, option lists, and default wording.
- `rules.py` - rule engine for missing information, warnings, considerations, and readiness classification.
- `study_recommender.py` - rule-based study pathway, procedure ethics considerations, 3Rs and animal timeline.
- `flowchart.py` - renders the recommended pathway as a PNG flowchart for the app and Word report.
- `ai_writer.py` - rule-based introduction and aims helper. It does not call AI services.
- `report_generator.py` - structured report assembly and Word document generation.
- `data_model.py` - empty response template and worked example data for tests.
- `tests/` - basic automated tests.
- `requirements.txt` - Python packages needed to run and test the app.

The structure is intentionally modular so later study modules can be added for PK, biodistribution, tumour efficacy, RNA/LNP, or procedure development studies without putting all logic into `app.py`.

## Run in a web browser with Streamlit Community Cloud

1. Go to https://share.streamlit.io/ and sign in with GitHub.
2. Deploy this repository: `anthonywignall-eng/animal-ethics-study-design-assistant`.
3. Use branch `main`.
4. Use main file path `app.py`.
5. Streamlit Cloud will install packages from `requirements.txt`.

After deployment, Streamlit gives you a shareable `.streamlit.app` web link.

If the app is already deployed, use **Reboot** or **Redeploy** in Streamlit Cloud after these repository changes so it pulls the latest code.

## Install Python for local running

If Python is not already installed, install Python 3.10 or newer from:

https://www.python.org/downloads/

On Windows, tick **Add Python to PATH** during installation if the installer offers that option.

## Run locally on Windows PowerShell

Open PowerShell, then move into the repository folder:

```powershell
cd path\to\animal-ethics-study-design-assistant
```

Create a project-specific Python environment:

```powershell
py -m venv .venv
```

Activate it:

```powershell
.\.venv\Scripts\Activate.ps1
```

If PowerShell blocks activation, run this once in the same terminal and then activate again:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\.venv\Scripts\Activate.ps1
```

Install the dependencies into that environment:

```powershell
py -m pip install --upgrade pip
pip install -r requirements.txt
```

Start the app:

```powershell
streamlit run app.py
```

Streamlit should open the app in your browser. If it does not, open:

```text
http://localhost:8501
```

## Run tests

After installing dependencies:

```bash
pytest -q
```

The tests cover selected rule triggers, study recommendation behaviour, flowchart rendering and Word report generation from example response sets.

## How to modify questions and rules later

1. Add options or stage labels in `config.py`.
2. Add the input field in the appropriate render function in `questionnaire.py`.
3. Add missing-information checks or warnings in `rules.py`.
4. Add recommendation pathway logic, procedure ethics text or timeline steps in `study_recommender.py`.
5. Add the field to summaries or report tables in `report_generator.py` if it should appear in the final report.
6. Add or update tests in `tests/`.

## Privacy limitations

Version 1 is session-only and does not intentionally save questionnaire responses. The app does not send responses to external AI or API services. Users should still avoid entering unnecessary personal information, confidential project identifiers, unpublished proprietary details or sensitive data until the intended hosting environment has been reviewed.

## Recommended next steps before real institutional use

- Review wording with the Animal Ethics Committee office, veterinarian and relevant investigators.
- Confirm institution-specific humane endpoint, CO2/euthanasia, blood collection, anaesthesia/analgesia and monitoring language.

- Add user acceptance testing with animal research assistants and project leads.
- Add versioning, audit review and controlled deployment if the tool becomes part of a formal workflow.
